Agent Evaluation

Agentic Perception and Routing Evaluation

DexHoldem evaluates whether an embodied agent can parse a real tabletop state into the structured memory needed by the system router: waiting, recovery, showdown judgment, legal decision states, and primitive dispatch all depend on this parse.

Bench Definition

The benchmark isolates perception from poker strategy and physical execution.

Each problem corresponds to one tabletop state sampled from a real game trajectory. The perceiver receives the current agent-view capture plus predecessor-state context when it is relevant, then writes a structured state that can be checked by a deterministic evaluator.

36 Problems

Fixed states cover turn gates, robot progress, held-card reads, cached sequence continuation, recovery, table decisions, fold wins, showdown, and collect-winnings.

Single Current Frame

The sampled state's agent-view image is the only current frame parsed from raw pixels; previous states are context rather than new visual targets.

Routing-Relevant Output

The output is not a caption. It is structured game-state memory used by wait gates, recovery gates, legal action selection, and terminal outcome handling.

Embodied System Context

The evaluated parse is the front end of a rule-routed agent loop.

In the full embodied system, each loop iteration captures an agent-view image, parses it into structured memory, routes through deterministic workflow gates, and dispatches a high-level agent primitive only when a new decision is needed.

Loop Stages

acting, atom_idle, idle, win, lose, to_recover, and down decide whether to wait, continue, recover, stop, or ask for help.

Rule-Based Router

The router handles waiting, primitive verification, continuation of cached multi-atom translations, recovery, and human-help escalation before prompting the main agent.

Agent Sandbox

The runtime bundles workflow rules, visual guidelines, capture and state helpers, router scripts, action translation, robot dispatch, and audio side effects.

Evaluation Protocol

Each perceiver is evaluated inside its native agent harness.

Codex, Claude Code, and Gemini CLI are given the same agent-view observation, predecessor-state context when relevant, visual guidelines, and workflow guidelines. The submitted structured state is scored by strict exact match and field-wise diagnostic columns.

Result Takeaways

Complete routing-relevant state recovery remains unreliable.

Strict Overall

Overall is strict problem-level exact match. The best strict score is 34.3%, reached by Opus 4.7 in the Claude Code harness.

Field-Wise Avg

GPT 5.5 reaches the best unweighted field-wise average at 66.8%, while its strict Overall remains 31.5%.

Chip Bottleneck

Current bet chips and opponent chip inventory are the weakest routing-critical fields, peaking at 45.8% and 43.8%.

Failure Modes

Strong isolated fields still fail to compose into full state recovery.

Blind information is nearly saturated and turn ownership is often reliable, but table decision and outcome-judge states require many fields to be correct simultaneously. Exact chip dictionaries are especially brittle because small stacked chips are often partially occluded on the far side of the table.
