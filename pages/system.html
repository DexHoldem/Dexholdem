DexHoldem Skills

A coding-agent native runtime for capture, perception, routing, execution, verification, recovery, and auditable state progression.

Runtime Contract

  1. CaptureRead s_current/00_capture.jpg from the experiment folder.
  2. ParseWrite 01_parsed_state.md with loop stage, robot behavior, table fields, and uncertainties.
  3. RouteRun router gates such as visual_parse, resolve_turn, resolve_uncertain_fields, and verify_sequence_complete.
  4. ExecuteTranslate high-level actions into supported card or chip primitive commands.
  5. VerifyCompare pre/post observations, update retry counters, and decide continue, recover, or request human help.
  6. AdvanceCreate the next state directory and synchronize durable caches.

Runtime States

Example state folders from DexHoldem Skills.

Each state directory stores a capture image plus parsed-state, action, cache, and verification artifacts that make the agent loop inspectable after a run.

Skill Variants

The released Skills package now includes native, isolated-perception, and backend-stream variants for different agent/runtime tradeoffs.

State Files

The experiment stores captured images, parsed states, action records, hole_card_cache.json, and action_sequence.json for replayable inspection.

Monitor Dashboard

The monitor watches the experiment directory and displays current capture, loop stage, table state, safety counters, and human-help alerts.
